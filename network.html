一、概述

1. 计算机网络的定义 ⭐

计算机网络是指将分布在不同地理位置，具有独立功能的许多计算机用传输介质连接起来，按照某种协议进行通信，并通过网络操作系统进行管理实现资源共享的系统。

根本目的：实现资源共享

2. 互联网的组成

部分组成作用
边缘部分所有连接在互联网上的主机（端系统）用户直接使用，进行通信和资源共享
核心部分大量网络和路由器为边缘部分提供连通性和交换服务

3. 互联网基础结构发展的三个阶段 ⭐

  1. 第一阶段（1969-1990）：ARPANET → 1983年TCP/IP成为标准协议 → 1983年是互联网诞生时间
  2. 第二阶段（1985-1993）：NSFNET三级结构（主干网、地区网、校园网）
  3. 第三阶段（1993-现在）：ISP多层次结构（主干ISP、地区ISP、本地ISP）

6. 时延的四种组成 ⭐

  • 发送时延 = 数据帧长度 / 发送速率（发生在机器内部）
  • 传播时延 = 信道长度 / 电磁波传播速率（发生在传输媒体上）
  • 处理时延：分析首部、差错检验、查找路由
  • 排队时延：在路由器队列中排队等待

总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延

9. 网络协议三要素

  • 语法：数据与控制信息的结构或格式
  • 语义：需要发出何种控制信息，完成何种动作
  • 同步：事件实现顺序的详细说明

10. 端系统通信方式

  • C/S方式（客户-服务器）：客户是服务请求方，服务器是服务提供方
  • P2P方式（对等连接）：每个主机既是客户又是服务器

二、物理层

1. 物理层的四大特性 ⭐

  • 机械特性：接口接线器的形状、尺寸、引线数目等
  • 电气特性：各条线上电压的范围
  • 功能特性：某条线上某一电平电压的意义
  • 过程特性：不同功能的各种可能事件的出现顺序

2. 数据通信基本概念 ⭐

  • 消息：如话音、文字、图像、视频
  • 数据：运送消息的实体
  • 信号：数据的电气或电磁表现
    • 模拟信号：取值连续
    • 数字信号：取值离散
  • 码元：代表不同离散数值的基本波形

3. 三种通信方式

  • 单工通信：单向传输
  • 半双工通信：双向交替传输
  • 全双工通信：双向同时传输

4. 基本调制方法

  • 调幅 (AM)：载波振幅随基带信号变化
  • 调频 (FM)：载波频率随基带信号变化
  • 调相 (PM)：载波初始相位随基带信号变化

5. 奈氏准则 (Nyquist) ⭐

在带宽为 W (Hz) 的低通信道中，若不考虑噪声影响，码元传输的最高速率 = 2W（码元/秒）

意义：激励工程人员探索更先进的编码技术，使每个码元携带更多比特信息量。

6. 香农公式 (Shannon) ⭐

C = W log₂(1 + S/N) (bit/s)

  • W：信道带宽 (Hz)；S/N：信噪比
  • 信噪比(dB) = 10 log₁₀(S/N)
  • 意义：告诫工程人员，无论多么复杂的编码，都不可能突破信息传输速率的绝对极限

7. 传输媒体 ⭐

类型媒体特点
导引型双绞线绞合度越高，可用速率越高；UTP(无屏蔽)/STP(屏蔽)；超5类可达1Gbit/s
同轴电缆抗干扰好，传输较高速率
光纤容量大、损耗小、抗电磁干扰、保密性好
多模光纤：LED光源，适合近距离
单模光纤：激光光源，适合远距离(几百公里)
非导引型无线电微波直线传播，地球表面约50km；多径效应；卫星通信传播时延大(250~300ms)

8. 信道复用技术 ⭐

技术原理
频分复用 FDM不同用户占用不同频带，同一时间
时分复用 TDM不同用户占用不同时隙，同一频带
统计时分复用 STDM按需动态分配时隙，提高利用率
波分复用 WDM光的频分复用
码分复用 CDM各用户使用不同码型，同一时间同一频带

9. 宽带接入技术

  • ADSL（非对称数字用户线）：下行>>上行，利用现有电话线
  • FTTx（光纤到...）：FTTH（光纤到户）、FTTB（大楼）、FTTC（路边）
  • PON（无源光网络）：EPON和GPON

三、数据链路层

1. 数据链路层信道类型

  • 点对点信道：一对一的通信方式
  • 广播信道：一对多的广播通信方式

2. 三个基本问题 ⭐

  • 封装成帧：在一段数据前后添加首部和尾部，构成帧。MTU规定数据部分长度上限
  • 透明传输：无论什么比特组合都能原样通过。解决方法：字节填充（异步）/零比特填充（同步）
  • 差错控制：使用CRC循环冗余检验

3. CRC循环冗余检验 ⭐

原理：在数据M后面添加n位冗余码（FCS），使得接收端用除数P除后余数为0。

  • CRC是一种检错方法，FCS是冗余码
  • CRC只能做到无差错接受，但这不是可靠传输
  • 可靠传输还需要：帧编号、确认和重传机制

常用生成多项式：CRC-16、CRC-CCITT、CRC-32

4. PPP协议（点对点协议）⭐

  • 三个组成部分：封装方法 + LCP（链路控制协议）+ NCP（网络控制协议）
  • PPP帧格式：标志7E + 地址FF + 控制03 + 协议 + 信息 + FCS + 标志7E
  • 透明传输：异步用字节填充，同步用零比特填充（5个连1后插入0）

5. 局域网数据链路层（两个子层）

  • LLC子层（逻辑链路控制）：与传输媒体无关
  • MAC子层（媒体接入控制）：与传输媒体有关

6. CSMA/CD协议 ⭐

载波监听多点接入/碰撞检测

  • 多点接入：总线型网络
  • 载波监听：发送前后都在检测信道
  • 碰撞检测：边发送边检测电压变化

关键参数：

  • 争用期（碰撞窗口）= 2τ = 51.2 μs（10 Mbit/s以太网）
  • 最短有效帧长 = 64字节（争用期内可发送512 bit）
  • 最大端到端长度 ≈ 5 km

7. 截断二进制指数退避算法 ⭐

  • 基本退避时间 = 2τ
  • r 从 [0, 1, …, 2ᵏ-1] 中随机选（k = Min[重传次数, 10]）
  • 重传时延 = r × 基本退避时间
  • 重传16次仍失败→丢弃帧，向高层报告

8. MAC地址

  • MAC地址（硬件地址/物理地址）：48位，固化在适配器ROM中
  • 帧的目的地址与适配器硬件地址一致时，才接收该帧

9. 以太网标准

  • DIX Ethernet V2 和 IEEE 802.3（两个标准可互操作）
  • 采用曼彻斯特编码（频带宽度增加一倍）
  • 无连接工作方式，不编号不确认

四、网络层

1. 网络层提供的两种服务 ⭐

对比方面虚电路服务数据报服务
思路可靠通信由网络保证可靠通信由用户主机保证
连接建立必须不需要
终点地址仅连接建立阶段使用每个分组都有完整地址
分组转发同一路由独立选择路由
分组顺序按序到达不一定按序
差错/流量控制可由网络负责由用户主机负责

互联网采用数据报服务。

2. IP地址分类 ⭐

类别前缀网络号主机号范围最大主机数
A01字节3字节1.0.0.0~126.0.0.016777214
B102字节2字节128.1.0.0~191.255.0.065534
C1103字节1字节192.0.1.0~223.255.255.0254
D1110多播地址
E1111保留

特殊IP地址：网络号127为环回测试；主机号全0为网络地址；主机号全1为广播地址。

3. 无分类编址CIDR ⭐

  • 斜线记法：a.b.c.d/n，前n位是网络前缀
  • 地址掩码（子网掩码）：一连串1 + 一连串0，1的个数 = 网络前缀长度
  • 网络地址 = IP地址 AND 地址掩码
  • 路由聚合：多个CIDR地址块合并为一个，减少转发表

常用CIDR地址块：/24→256地址, /16→64K地址, /8→16M地址

三个特殊CIDR地址块：/32（主机路由）、/31（点对点链路）、/0（默认路由）

4. IP数据报格式

  • 首部：固定20字节 + 可变选项（最长60字节）
  • TTL（生存时间）：每经过一个路由器减1，为0时丢弃
  • 协议字段：6=TCP, 17=UDP, 1=ICMP
  • 分片：标识+标志+片偏移，用于IP数据报分片和重组

5. IP地址与MAC地址

  • IP地址：网络层地址，逻辑地址，端到端不变
  • MAC地址：数据链路层地址，硬件地址，逐跳变化
  • ARP（地址解析协议）：IP→MAC地址映射
  • ARP缓存：存储IP-MAC映射，有过期时间

6. ICMP（网际控制报文协议）

  • ICMP差错报告报文：终点不可达、时间超过、参数问题、改变路由
  • ICMP询问报文：回送请求和回答（PING）、时间戳请求和回答
  • PING：测试连通性，使用ICMP回送请求和回答报文
  • tracert：跟踪路由，利用TTL和ICMP时间超过报文

7. 路由选择协议 ⭐

分类协议算法说明
内部网关协议 IGPRIP距离向量跳数（最多15跳），周期性广播
内部网关协议 IGPOSPF链路状态Dijkstra算法，快速收敛
外部网关协议 EGPBGP路径向量不同AS之间，基于策略

8. IPv6

  • 地址长度：128位（vs IPv4的32位）
  • 地址表示：8组16进制，冒号分隔（如2001:0db8:...）
  • 过渡技术：双协议栈、隧道技术

9. VPN和NAT

  • VPN（虚拟专用网）：通过公用网络建立专用网络连接
  • NAT（网络地址转换）：私有地址↔公网地址转换，缓解IPv4地址枯竭

1. 运输层的作用

向高层用户屏蔽下面网络核心的细节，使应用进程看见端到端的逻辑通信信道。

2. UDP vs TCP ⭐

特性UDPTCP

面向面向报文（不拆分不合并）面向字节流

3. 端口号 ⭐

类型范围说明
熟知端口0~1023IANA分配，如 HTTP(80)、DNS(53)、SMTP(25)、FTP(21/20)、SSH(22)
登记端口1024~49151在IANA登记
短暂端口49152~65535客户端使用，通信结束后回收

4. 停止等待协议 ⭐

  • 每发一个分组就停止，等待确认后再发下一个
  • 超时重传：计时器超时未收到确认→重传
  • 编号：防止重复和确认混淆
  • 缺点：信道利用率低

5. 连续ARQ协议（流水线传输）

  • 发送窗口：窗口内分组连续发送，不需等待确认
  • 累积确认：确认号N表示N-1及之前的全收到
  • Go-back-N：出现差错时回退N个分组重传

6. TCP报文段首部字段 ⭐

  • 序号(seq)：本报文段第一个字节的序号
  • 确认号(ack)：期望收到对方下一个报文段的第一个字节序号
  • 数据偏移：TCP首部长度（单位：4字节）
  • 控制位：SYN(同步)、FIN(释放)、ACK(确认)、RST(复位)、URG(紧急)、PSH(推送)
  • 窗口：接收方允许发送方的发送窗口大小（流量控制）

7. TCP拥塞控制 ⭐

四种算法：

算法规则
慢开始cwnd从1开始，每收到一个ACK加1（指数增长），直到ssthresh
拥塞避免每经过一个RTT，cwnd加1（线性增长，加法增大AI）
快重传连续收到3个重复ACK→立即重传丢失报文段
快恢复收到3个重复ACK→ssthresh=cwnd/2, cwnd=ssthresh, 进入拥塞避免（乘法减小MD）

发送窗口上限值 = Min [rwnd, cwnd]

当超时重传：ssthresh = max(cwnd/2, 2), cwnd = 1, 重新慢开始

8. TCP三次握手 ⭐

客户端                             服务器
  |-------- SYN=1, seq=x ---------->|
  |<------- SYN=1, ACK=1,          |
  |         seq=y, ack=x+1 --------|
  |-------- ACK=1, seq=x+1,       |
  |         ack=y+1 --------------->|
  |                                  |
双方进入ESTABLISHED状态
  • 第一次：客户端→服务器 SYN报文段（SYN=1, seq=x），不携带数据但消耗一个序号
  • 第二次：服务器→客户端 确认报文段（SYN=1, ACK=1, seq=y, ack=x+1），不携带数据消耗序号
  • 第三次：客户端→服务器 ACK报文段（ACK=1, seq=x+1, ack=y+1），可携带数据

9. TCP四次挥手 ⭐

客户端                             服务器
  |------ FIN=1, seq=u ------------>|   （客户端主动关闭）
  |<------ ACK=1, seq=v,           |
  |         ack=u+1 ----------------|   （半关闭状态）
  |<------ FIN=1, ACK=1,           |
  |         seq=w, ack=u+1 --------|   （服务器也关闭）
  |------ ACK=1, seq=u+1,         |
  |         ack=w+1 --------------->|   （TIME-WAIT等待2MSL）
  |                                  |
  • FIN报文段即使不携带数据，也消耗一个序号
  • TIME-WAIT状态持续2MSL（最长报文段寿命）

六、应用层

1. DNS域名系统 ⭐

  • 作用：域名→IP地址转换
  • 结构：层次树状结构（根→顶级域→二级域→...）
  • 通信：使用UDP，端口53

四种域名服务器：

类型作用
根域名服务器最高层，知道所有顶级域名服务器地址（全球13套装置）
顶级域名服务器管理该顶级域下的所有二级域名
权限域名服务器负责一个区的域名解析
本地域名服务器每个ISP/大学都有，主机首先查询的服务器

2. 文件传送协议 FTP

  • 使用TCP连接，端口21（控制连接）+ 20（数据连接）
  • 提供交互式访问，允许客户指定文件类型和格式

3. 万维网 WWW ⭐

  • 组成：统一资源定位符URL + 超文本传送协议HTTP + 超文本标记语言HTML
  • URL格式：协议://主机:端口/路径
  • HTTP：使用TCP，端口80

HTTP报文：

  • 请求报文：方法(GET/POST/PUT/DELETE) + URL + 版本 + 首部行 + 实体主体
  • 响应报文：版本 + 状态码 + 短语 + 首部行 + 实体主体
  • 常见状态码：200 OK, 301永久移动, 404 Not Found, 500服务器错误
  • HTTP是无状态协议，使用Cookie跟踪用户

4. 电子邮件 ⭐

协议作用端口

MIME邮件扩充（支持非ASCII文本、多媒体等）—

5. DHCP动态主机配置协议 ⭐

  • 作用：自动分配IP地址、子网掩码、默认网关、DNS服务器等
  • 使用UDP，服务器端口67，客户端端口68
  • 流程：DHCP发现→DHCP提供→DHCP请求→DHCP确认（DISCOVER→OFFER→REQUEST→ACK）

七、网络安全

1. 网络安全威胁 ⭐

攻击类型方式是否可检测
被动攻击截获（窃听通信内容）、流量分析通常检测不出
主动攻击篡改、恶意程序（病毒/蠕虫/木马）、拒绝服务DoS可检测
  • DDoS（分布式拒绝服务）：成百上千网站集中攻击一个网站
  • 被动攻击→加密技术；主动攻击→加密+鉴别技术

2. 网络安全的四个目标

  • 保密性：只有收发双方能理解信息内容
  • 端点鉴别：鉴别发送方和接收方的真实身份
  • 信息完整性：信息内容未被篡改
  • 运行安全性：系统能正常运行并提供服务（访问控制）

3. 两类密码体制 ⭐

特性对称密钥密码体制公钥密码体制
密钥加密密钥 = 解密密钥加密密钥(公钥) ≠ 解密密钥(私钥)
代表DES(56位), 3DES, AES(128/192/256位)RSA
优点速度快密钥分配方便，支持数字签名
缺点密钥分配困难速度慢

4. 数字签名

  • 功能：报文鉴别（证明来源）、不可否认（发送方不能否认）、完整性（防篡改）
  • 原理：发送方用私钥加密→数字签名；接收方用公钥解密验证

5. 防火墙与入侵检测

  • 防火墙：控制网络访问，分为分组过滤和应用网关
  • 入侵检测系统 IDS：检测异常网络活动

八、互联网上的音频/视频服务

核心概念

  • 流式存储：边下载边播放（如YouTube）
  • 实时流式：实时传输协议 RTP + RTCP + RTSP
  • RTP（实时传输协议）：提供时间戳和序号，使用UDP
  • RTCP（实时传输控制协议）：提供QoS反馈
  • QoS（服务质量）：延迟、抖动、丢包率等指标

九、无线网络和移动网络

核心概念

  • 无线局域网 WLAN：使用IEEE 802.11标准（Wi-Fi）
  • CSMA/CA：载波监听多点接入/碰撞避免（无线不能用CSMA/CD）
  • ISM频段：2.4GHz和5GHz，可自由使用
  • 蜂窝移动网络：2G→3G→4G(LTE)→5G
  • 移动IP：支持主机在不同网络间移动时保持通信

简述电路交换、报文交换和分组交换的主要区别。

题4 计算题 🔥

一个信道的带宽为3kHz，信噪比为30dB。求该信道的最大数据传输速率（bit/s）。

解答：
第一步：30dB → S/N：30 = 10log₁₀(S/N)，S/N = 1000
第二步：香农公式 C = W log₂(1+S/N) = 3000 × log₂(1+1000) ≈ 3000 × 9.97 ≈ 29910 bit/s ≈ 30 kbit/s

题5 计算题 🔥

若信道带宽为4kHz，采用4相调制技术（每个码元携带2比特）。不考噪虑声，求最大数据传输速率。

解答：
奈氏准则：码元最高速率 = 2W = 2×4000 = 8000 Baud
每码元2比特：最大数据率 = 8000 × 2 = 16000 bit/s = 16 kbit/s

题8 计算题 🔥

已知发送数据M=101001，除数P=1101。求CRC冗余码和发送的数据。

解答：
① M后加3个0（P是4位，n=3）：101001000
② 模2除法：101001000 ÷ 1101
③ 余数 = 001（3位冗余码）
④ 发送数据 = 101001001

题9 简答题

简述CSMA/CD和CSMA/CA的主要区别，为什么无线局域网不能使用CSMA/CD？

答案要点：①CSMA/CD通过电压检测碰撞，有线可实现同时收发；②无线信号自己发送时远强于接收，无法检测碰撞（实现费用大）；③隐蔽站问题：两个站点检测不到彼此，同时向AP发→碰撞；④CSMA/CA采用碰撞避免+RTS/CTS预约机制。

题12 简答题 🔥

画出TCP三次握手的过程图，标注每一次的SYN、ACK、seq、ack的值及状态变化。

答案要点：
① 客户端→服务器：SYN=1, seq=x（CLOSED→SYN-SENT）
② 服务器→客户端：SYN=1, ACK=1, seq=y, ack=x+1（LISTEN→SYN-RCVD）
③ 客户端→服务器：ACK=1, seq=x+1, ack=y+1（SYN-SENT→ESTABLISHED）
④ 服务器收到③后：SYN-RCVD→ESTABLISHED

题13 简答题

简述TCP四次挥手释放连接的过程，并说明为什么需要TIME-WAIT状态。

答案要点：四次挥手过程（略，见考点）；TIME-WAIT持续2MSL：①保证最后一个ACK能到达服务器 ②防止已失效的连接请求报文出现在新连接中。
