一、HTML（超文本标记语言）

1. Web标准三大组成部分

  • HTML：负责网页的结构（页面元素和内容）
  • CSS：负责网页的表现（外观、位置、颜色、大小等样式）
  • JavaScript：负责网页的行为（交互效果）

2. HTML基本概念

  • HTML（HyperText Markup Language）：超文本标记语言
  • HTML标签预定义，由浏览器直接解析运行
  • 特点：简易性、可扩展性、与平台无关、跨网络超文本链接

3. HTML5文档基本结构 ⭐

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <!-- 页面内容 -->
</body>
</html>

4. HTML元素与标签

  • 成对标签：<标签名> 内容 </标签名>（大多数）
  • 自闭合标签：<br>、<img>、<input>、<link>、<hr>（空元素，无内容）
  • 标签大小写不敏感，建议小写；标签可嵌套
  • 注释：<!-- 注释内容 -->

5. HTML属性 ⭐

  • 格式：<标签名 属性名="属性值"> 内容 </标签名>
  • 属性值始终用引号（双引号或单引号均可）
  • 全局属性（适用于所有元素）：
    • class：指定类名，用于CSS/JS
    • id：唯一标识符，用于CSS/JS
    • style：内联CSS样式

6. DOCTYPE声明

  • <!DOCTYPE html> 是HTML5文档类型声明，必须放在最顶部
  • 告诉浏览器使用HTML5标准解析，避免"怪异模式"
  • HTML5声明比HTML4/XHTML更简洁

7. lang属性

  • <html lang="en">（英文）/ <html lang="zh">（中文）

8. head标签内容

  • <title>：页面标题
  • <meta>：元数据（name、content、http-equiv属性）
  • <link>：引入外部资源（如CSS）
  • <style>：内部CSS样式

9. meta标签常用方式 ⭐

<meta charset="UTF-8">          <!-- 字符编码 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">  <!-- 视口/响应式 -->
<meta http-equiv="refresh" content="5;url=https://example.com/">           <!-- 5秒后跳转 -->

10. body标签属性（已过时，CSS替代）

属性说明CSS替代
bgcolor背景颜色background-color
text文本颜色color
link未访问链接颜色a:link { color: ... }
vlink已访问链接颜色a:visited { color: ... }
alink活动链接颜色a:active { color: ... }
background背景图片background-image

11. 颜色表示

  • 英文名称：red、blue、green等
  • 十六进制RGB：#FF0000（红）、#000000（黑）、#FFFFFF（白）

12. 标题标签

  • <h1> ~ <h6>，6级标题，h1最大、h6最小，显示为粗体
  • align属性（已过时）：left | center | right

13. 文字样式标签

  • <font face="字体" size="1-7" color="颜色">（已过时，建议用CSS）
  • 物理样式：<b>粗体、<i>斜体、<u>下划线、<s>删除线、<sup>上标、<sub>下标、<big>大字体、<small>小字体
  • 逻辑样式：<strong>重点突出(粗体)、<em>强调(斜体)、<address>地址、<code>代码

14. 特殊字符 ⭐

字符转义码字符转义码
<&lt;"&quot;
>&gt;&&amp;
空格&nbsp;©&copy;

15. 段落与换行

  • <p>：段落标签，段前段后空一行（结束标签可省略但不推荐）
  • <br>：换行标签，不插入空行
  • <nobr>：禁止换行
  • <hr>：水平线（可设size、width、align、color属性）

16. 多媒体标签 ⭐

图像：<img src="URL" alt="文字说明" width="宽" height="高">（自闭合）

  • src和alt是必需属性
  • 属性：align（对齐）、border（边框，0为无边框）

视频：

<video width="320" height="240" controls="controls">
    <source src="movie.mp4" type="video/mp4" />
    <source src="movie.ogg" type="video/ogg" />
</video>
  • 属性：autoplay、controls、loop、muted、poster、preload
  • HTML5支持三种视频格式：MP4、WebM、OGG

17. 超链接 ⭐

<a href="URL" target="_blank|_self" title="提示文字">链接</a>
  • href：目标URL（#回到顶部）
  • target="_self"：当前窗口打开（默认）
  • target="_blank"：新窗口打开
  • title：鼠标悬停提示

18. 块级元素 vs 行内元素 ⭐

特性块级元素行内元素
占行独占一行不独占一行
宽度默认父容器100%由内容决定
宽高设置可以无效
常见元素<div>、<p>、<h1>-<h6>、<ul>、<li>、<section><span>、<a>、<strong>、<img>
  • <div>：块级无语义元素，用于布局和分组
  • <span>：行内无语义元素，用于样式化文本
  • <div>可包含块级和行内元素；<span>只能包含行内元素

19. HTML5语义元素

  • <header>：页面/区块头部
  • <nav>：导航区域
  • <main>：页面核心内容（唯一）
  • <section>：主题性区块
  • <article>：独立完整的内容块
  • <aside>：侧边栏/辅助内容
  • <footer>：页面/区块底部

20. 表格 ⭐

<table border="1">
    <tr><th>表头1</th><th>表头2</th></tr>
    <tr><td>数据1</td><td>数据2</td></tr>
</table>
  • colspan="n"：跨n列合并（水平合并）
  • rowspan="n"：跨n行合并（垂直合并）
  • cellpadding：单元格内边距 / cellspacing：单元格间距

21. 列表

  • 无序列表：<ul type="disc|circle|square"><li>...</li></ul>
  • 有序列表：<ol type="1|A|a|I|i"><li>...</li></ol>
  • <li>必须嵌套在<ul>或<ol>中使用

22. 表单 ⭐

<form action="提交URL" method="GET|POST">
    <input type="text|password|radio|checkbox|submit|button|file|hidden" name="名称" value="值">
    <select name="名称"><option value="值">选项</option></select>
    <textarea name="名称"></textarea>
</form>
  • GET：数据拼接在URL后(?key=value)、有大小限制、可收藏缓存
  • POST：数据在请求体中、理论上无限制、较安全
  • radio：同name互斥；checkbox：同name组提交或多name分别提交
  • <label for="id">：为input添加可点击标签

二、CSS（层叠样式表）

1. CSS概念

  • CSS（Cascading Style Sheets）：层叠样式表/级联样式表
  • 不是编程语言，是样式语言；文件后缀.css
  • 作用：改变浏览器默认风格、内容与样式分离、同一内容不同样式

2. CSS基本语法

选择器 {
    属性: 值;
    属性: 值;
}
  • 选择器+一条或多条声明组成；每条声明以分号结尾
  • CSS注释：/* 注释内容 */

3. CSS引入方式（三种）⭐

引入方式写法优先级使用频率
行内样式<h1 style="color:red">最高较少
内部样式表<style>h1{color:red}</style>中较高
外部样式表<link rel="stylesheet" href="xxx.css">中最高
  • <link>只需rel和href属性，无需type属性
  • 不推荐@import（无法并行下载，增加加载时间）

4. CSS冲突处理 ⭐

  • 优先级：行内样式 > 内部样式表 = 外部样式表
  • 同优先级：后定义的样式生效（类似穿衣服，最后穿的别人能看到）
  • !important可强制提升优先级（不推荐滥用）

5. 选择器权重（Specificity）⭐

选择器语法权重说明
通配符选择器*{}0选取所有元素
元素选择器p{}1根据标签名
类选择器.red{}10.开头，类名不能数字开头
ID选择器#para1{}100#开头，id页面唯一
内联样式style=""1000直接在标签上

优先级顺序：行内样式 > ID选择器 > 类选择器 > 元素选择器 > 通配符选择器

巧记：越精准的设置优先级越高。

6. 组合选择器

选择器符号示例说明
后代选择器空格div pdiv内所有p（任意层级）
子选择器>div>pdiv直接子元素p
接续兄弟选择器+div+pdiv后紧邻的第一个同级p
后续兄弟选择器~div~pdiv后所有同级p

7. 伪类选择器 ⭐

锚伪类（必须按LVHA顺序！）

a:link    { color: black; }   /* L - 未访问链接 */
a:visited { color: green; }   /* V - 已访问链接 */
a:hover   { color: red; }     /* H - 鼠标悬停 */
a:active  { color: blue; }    /* A - 点击激活 */
  • 其他伪类：:first-child、:last-child、:nth-child(n)、:focus、:checked

8. 伪元素选择器

  • ::before：元素之前插入内容（必须设content属性）
  • ::after：元素之后插入内容
  • ::first-letter：首字母
  • ::first-line：首行
  • ::selection：用户选中的部分

伪类 vs 伪元素：伪类用:修饰已有元素状态；伪元素用::创建虚拟元素（CSS3以前用:）

9. 属性选择器

  • a[title]：有title属性的a元素
  • a[href="https://example.org"]：href精确匹配的a元素

10. 盒子模型（Box Model）⭐

从外到内：margin（外边距）→ border（边框）→ padding（内边距）→ content（内容）

  • padding：1~4个值（上、右、下、左，顺时针）；不接受负值
  • margin：1、2、4个值；margin: auto可水平居中
  • border：border: 宽度 样式 颜色，如border: 5px solid red

11. 文档流

  • 标准流：块元素自上而下，行内元素从左到右
  • display: block：块级盒子，前后换行
  • display: inline：行级盒子，不换行
  • 脱离文档流三种方式：浮动、绝对定位、固定定位

12. 浮动布局（Float）

  • float: left | right | none
  • 浮动是相对于父元素移动
  • 浮动坍塌：父容器内所有子元素浮动后，父容器高度变为0
  • 解决：给父容器设置overflow: hidden或使用clearfix

13. 弹性盒子（Flex Box）⭐

display: flex;
flex-direction: row | column | row-reverse | column-reverse;
justify-content: center | flex-start | flex-end | space-between | space-around;
align-items: center | stretch | flex-start | flex-end;

上下左右居中：{ justify-content: center; align-items: center; }

Flex是一维布局（一行或一列），二维布局用Grid。

14. 元素定位（Position）

值说明是否脱离文档流
static默认值，标准流否
relative相对自身原始位置偏移否（占据原位置）
absolute相对最近已定位祖先定位是
fixed相对浏览器窗口定位是
sticky滚动到指定位置变为fixed混合

15. CSS3 新特性

  • 圆角：border-radius: 20px;（≥50%产生圆形）
  • 阴影：box-shadow: h v blur spread color;
  • 渐变：linear-gradient(方向, 颜色1, 颜色2)
  • 变形：translate(x,y)、rotate(deg)、scale(x,y)
  • 过渡：transition: property duration timing-function delay;
  • 动画：@keyframes name { from{} to{} } + animation属性

三、JavaScript

1. JavaScript概述

  • JavaScript = ECMAScript（语法核心）+ DOM（文档对象模型）+ BOM（浏览器对象模型）
  • 1995年Brendan Eich发明，1997年成为ECMA标准
  • ES6（ECMAScript 2015）是最主流版本
  • JS是弱类型语言，声明时不需要指定数据类型

2. JS引入方式（三种）⭐

引入方式写法说明
内联脚本<button onclick="alert('Hello')">标签属性中，违反内容行为分离
内部脚本<script>alert('Hello')</script>HTML中，一般放在body底部
外部脚本<script src="demo.js"></script>⭐标签不能自闭合

3. JS书写语法

  • 区分大小写；分号可有可无（但多条同行必须分号）
  • 注释：//单行、/* */多行；大括号表示代码块
  • 输出：console.log()⭐（控制台）、window.alert()（警告框）、document.write()（写入HTML）

4. 变量声明 ⭐

关键字作用域重复声明说明
var全局/函数允许旧式声明，允许不赋初值
let块级不允许ES6新增，⭐官方推荐
const块级不允许声明常量，必须显式赋值

5. 基本数据类型（5种）⭐

类型说明示例
number64位双精度浮点（无单独整数类型）42, 3.14, NaN, Infinity
stringUTF-16字符串"hello", 'world', `template`
boolean布尔值true, false
undefined声明未赋值let x; → undefined
null空值（空对象引用）let y = null;
  • typeof null → "object"（JS历史遗留错误）
  • typeof NaN → "number"
  • NaN === NaN → false，用Number.isNaN()判断

6. 模板字符串（ES6）

const name = 'Tom', age = 30;

四、Servlet

1. 同步 vs 异步

  • 同步：顺序执行，阻塞式 → 页面卡死、不能渲染
  • 异步：彼此独立，非阻塞 → 界面响应流畅

2. 异步通信技术

  • Fetch API（2015）：浏览器原生，不需引入，不够方便
  • Axios（2016）：自动转换JSON，需手动导入js
  • async/await：async放函数前变异步；await等待完成，只阻塞当前函数
axios({ method:"get", url:"http://localhost:8080/item/selectAll" })
     .then(resp => { this.tableData = resp.data; });

3. HTTP请求

  • 请求方法：GET、POST、PUT、DELETE
  • URL格式：http://localhost:8080/login

4. JavaWeb三大组件

  • Servlet（服务连接器）：处理客户端请求
  • Filter（过滤器）：请求到达前拦截
  • Listener（监听器）：监听Web事件

5. Servlet概念

  • Servlet是JavaEE规范之一，本质上是一个接口
  • Servlet由Web服务器（Tomcat）创建和调用

6. Servlet接口方法

  • void init(ServletConfig config)：初始化，只执行一次
  • void service(ServletRequest req, ServletResponse resp)：处理请求，每次访问调用
  • void destroy()：销毁，释放资源时调用

7. Servlet生命周期（4个阶段）⭐

阶段说明执行次数
1. 加载和实例化默认第一次访问时创建（loadOnStartup=-1）；0或正整数→服务器启动时创建—
2. 初始化 init()加载配置、创建连接等初始化工作一次
3. 请求处理 service()每次请求Servlet时调用多次
4. 服务终止 destroy()释放内存或容器关闭时调用一次

8. HttpServlet ⭐

  • 继承层次：Servlet接口 → GenericServlet抽象类 → HttpServlet
  • 使用步骤：① 继承HttpServlet ② 重写doGet和doPost
  • 原理：获取请求方式，根据请求方式调用不同doXxx方法
  • GET请求参数在请求行中，POST请求参数在请求体中
@WebServlet("/demo1")
public class MyServlet extends HttpServlet {
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) { }
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) { }
}

9. Servlet urlPattern配置 ⭐

匹配模式配置示例说明
精确匹配@WebServlet("/user/select")精确路径
目录匹配@WebServlet("/user/*")匹配/user/下所有
扩展名匹配@WebServlet("*.html")注意开头无"/"
任意匹配@WebServlet("/") 或 /*兜底；/不匹配带扩展名

优先级：精确匹配 > 目录匹配 > /* > 扩展名匹配 > /

五、会话跟踪

1. 会话概念

  • HTTP是无状态协议，每个请求独立
  • 会话跟踪：维护浏览器状态，识别同一浏览器多次请求，在多次请求间共享数据
  • 使用场景：登录"记住我"功能

2. 会话跟踪技术

技术存储位置容量适用场景
Cookie客户端4KB通用会话跟踪
HttpSession服务端取决于服务器Java Web应用
JWT (Token)客户端—用户身份验证

3. Cookie ⭐

运行机制：服务器创建Cookie→响应发送→浏览器存储→后续请求自动携带→服务器读取

// 发送Cookie
Cookie cookie = new Cookie("username", "maxxie");
response.addCookie(cookie);

// 获取Cookie
Cookie[] cookies = request.getCookies();
for (Cookie c : cookies) {
    String name = c.getName();
    String value = c.getValue();
}

setMaxAge：

  • 正数：持久化到硬盘，到期自动删除
  • 负数（默认）：浏览器内存，关闭即销毁
  • 零：删除对应Cookie

缺点：安全风险、大小限制(4KB)、明文传输、用户可拒绝

4. Session ⭐（基于Cookie实现）

运行机制：服务器创建Session（唯一标识JSESSIONID）→以Cookie形式响应→浏览器存储→后续请求携带JSESSIONID→服务器查找Session

// 存储
HttpSession session = request.getSession();
session.setAttribute("username", "maxxie");

// 获取（false防止创建新session）
HttpSession session = request.getSession(false);
Object username = session.getAttribute("username");
  • 默认30分钟无操作自动销毁
  • session.setMaxInactiveInterval(60);：自定义失效时间
  • session.invalidate();：手动销毁（用户退出时使用）
  • 钝化：服务器正常关闭时写入硬盘
  • 活化：再次启动时加载到Session

5. Cookie vs Session对比 ⭐

特性CookieSession
存储位置客户端服务器端
存储类型字符串对象
安全性较低（易被篡改/读取）较高（仅Session ID）
数据大小单个不超过4KB理论上无限制
过期可设，关浏览器仍可存在关浏览器通常结束

六、Filter过滤器 & 请求转发与重定向

1. Filter（过滤器）⭐

  • 作用：拦截请求，完成通用操作
  • 应用场景：登录验证、跨域处理、统一编码、敏感字符过滤
  • 可拦截：Servlet请求、静态资源（HTML/CSS/JS）、图片等

实现步骤：① 定义类实现Filter接口 ② 重写init/doFilter/destroy ③ 配置拦截路径

生命周期：init（一次）→ doFilter（多次）→ destroy（一次）

2. 请求转发（forward）

  • 服务器内部的资源跳转，对客户端透明
  • 特征：1次请求、URL不变、共享request数据、仅限当前应用内
  • 实现：request.getRequestDispatcher("/path").forward(request, response)

3. 重定向（redirect）

  • 通知客户端去访问另一个资源（302+location）
  • 特征：2次请求、URL改变、不共享request数据、可任意URL
  • 实现：response.sendRedirect("/path")

4. 转发 vs 重定向 ⭐

特性转发（forward）重定向（redirect）
请求次数1次2次
URL变化不变改变
数据共享共享request不共享
目标限制仅当前应用内任意URL

5. 场景选择判断 ⭐

  • 需要共享request数据 → 转发
  • 跳转到外部网站 → 重定向
  • 避免表单重复提交 → 重定向
  • 不希望用户看到实际URL → 转发
  • 需要改变地址栏URL → 重定向

七、JSP（Java Server Pages）

1. JSP概念

  • JSP = HTML + Java，动态网页技术
  • JSP本质上就是一个Servlet（JSP→转换→.java文件→编译→.class文件）
  • 自动生成的Java文件继承HttpJspBase→继承HttpServlet

2. JSP基本语法 ⭐

元素语法位置用途
注释<%-- --%>任意服务器端，不发送客户端（与HTML注释<!-- -->不同）
脚本段<% %>_jspService()内嵌入Java逻辑代码
表达式<%= %>out.print()参数⭐输出（不能有分号！）
声明<%! %>类成员位置成员变量/方法（多请求共享）

关键区别：声明中变量是成员变量（多请求共享），脚本段中变量是局部变量（仅当前请求有效）

out（JspWriter）→ 客户端浏览器；System.out → 控制台

3. JSP指令标记

<%@ page contentType="text/html;charset=UTF-8" %>  <!-- 处理乱码 -->
<%@ page import="java.util.Date" %>                  <!-- 导入类 -->
<%@ page session="true" %>                           <!-- 启用session -->
<%@ include file="header.jsp" %>                      <!-- 静态包含 -->

4. JSP动作标记

<jsp:useBean id="MyBox" class="edu.hnucm.Box" scope="page" />
<jsp:getProperty name="MyBox" property="length" />
<jsp:setProperty name="MyBox" property="length" value="10" />
<jsp:include page="header.jsp" />  <!-- 动态包含 -->

Scope取值：page（当前页）→ request（请求期间）→ session（会话期间）→ application（整个应用生存期）

5. 静态包含 vs 动态包含 ⭐

特性<%@ include %>（静态）<jsp:include>（动态）
处理时机编译时运行时（每次请求）
效率高（只编译一次）稍低
变量作用域共享独立
修改反映需重新编译主文件立即反映

6. EL表达式

  • ${MyBox.length} 替代 <jsp:getProperty name="MyBox" property="length"/>
  • 不能直接赋值，赋值仍用setProperty

7. JSP优缺点

  • 优点：与Java生态集成、简化HTML输出、组件化
  • 缺点：前后端耦合严重、混杂交错、难维护
  • 现代替代：Thymeleaf（Spring推荐）、前后端分离（Spring Boot + Vue/React）

八、简单的Vue（前端框架）

1. Vue概述

  • Vue.js用于实现响应式数据绑定和组件化视图
  • 配合Element Plus（UI组件库）和Axios（HTTP请求）使用
  • Element Plus：https://element-plus.org/zh-CN/

2. 核心示例

<el-table-column prop="brandName" label="品牌名称">
<el-table-column prop="companyName" label="企业名称">

<script>
selectAll() {
    axios({
        method: "get",
        url: "http://localhost:8080/item/selectAll"
    }).then(resp => {
        this.tableData = resp.data;
    })
}
</script>

3. Vue核心概念

  • 数据绑定（{{ }} Mustache语法）
  • 事件处理
  • 组件化开发（Element Plus的el-table-column等）
  • 通过Axios与后端异步数据交互

九、项目思维（架构与开发模式）

1. C/S vs B/S架构

架构含义特点
C/SClient/Server（客户端/服务器）需要安装客户端
B/SBrowser/Server（浏览器/服务器）只需浏览器，易维护升级

2. 前后端分离开发 vs 混合开发

  • 前后端分离：前端(Vue/Element Plus/Nginx) + 后端(SpringBoot/MyBatis/MySQL)
  • 混合开发：前端代码和后端在同一项目中

3. Web工作流程

浏览器 → 前端服务器(前端程序) → 后端服务器(Java程序) → 数据库服务器

4. MVC三层架构

  1. 用户请求 → Controller（控制层）接收
  2. Controller调用Model（模型层）处理
  3. Model返回结果→Controller根据结果找相应View（视图）
  4. View渲染数据后响应给浏览器

5. 技术栈对比

体系前端后端服务器
JavaHTML/CSS/JS + Vue/Element PlusJSP/Servlet/SpringBootTomcat
.NetHTML/CSS/JSAspx/C#IIS

6. 课程技术路线

  • 前端：HTML/CSS/JS → Vue、Element Plus、Nginx
  • 后端：JSP/Servlet → Maven、SpringBoot、MySQL、MyBatis
  • 经典框架：Struts+Spring+Hibernate / Spring MVC+Spring+MyBatis

7. 前端技术发展

HTML+CSS+JS（打牢基础）→ jQuery（了解）→ Bootstrap/Vue.js（自学提升）

8. Web技术发展四阶段

  1. 纯Servlet（逻辑+页面拼接）
  2. Servlet + JSP（JSP仍嵌套Java）
  3. Servlet（逻辑）+ JSP（展示）→ MVC雏形
  4. Servlet（后端）+ HTML + Vue.js → 前后端分离
